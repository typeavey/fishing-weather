# 🎣 Fishing Weather Portal - Shell Scripts

## 📁 Remaining Scripts (Cleaned Up)

### **Core Scripts**

1. **`start-portal.sh`** - Quick start for local development
   - Activates virtual environment
   - Starts Flask app on localhost:5000
   - For development/testing

2. **`deploy.sh`** - Complete local deployment setup
   - Creates virtual environment
   - Installs dependencies
   - Initializes database
   - Creates startup and backup scripts
   - For initial setup

3. **`backup-database.sh`** - Database backup utility
   - Creates timestamped backups
   - Stores in `backups/` directory
   - For data protection

### **Production Scripts**

4. **`setup-production.sh`** - Production environment setup
   - Creates systemd service
   - Configures Apache reverse proxy
   - Sets up firewall rules
   - For Rocky Linux production deployment

5. **`update-deployment.sh`** - Production update script
   - Backs up database
   - Pulls latest changes
   - Updates dependencies
   - Restarts services
   - For production updates

## 🗑️ Removed Scripts

- ❌ `setup-systemd-service.sh` - Replaced by `setup-production.sh`
- ❌ `fix-rocky-server.sh` - Functionality merged into `setup-production.sh`
- ❌ `start-production.sh` - Redundant with systemd service
- ❌ `deploy-production.sh` - Replaced by `setup-production.sh`
- ❌ `setup-rocky-linux.sh` - Replaced by `setup-production.sh`
- ❌ `fishing-weather.service` - Generated by `setup-production.sh`

## 🎯 Usage Guide

### **Local Development**
```bash
# Initial setup
./deploy.sh

# Start portal
./start-portal.sh

# Backup database
./backup-database.sh
```

### **Production Deployment**
```bash
# First-time production setup
./setup-production.sh

# Future updates
./update-deployment.sh
```
